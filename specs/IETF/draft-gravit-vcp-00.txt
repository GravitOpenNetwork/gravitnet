Abstract
   Verifiable Convergence Protocol (VCP) defines the minimal data types
   and APIs required for autonomous agents and human-machine systems to
   achieve epistemic convergence without centralized truth arbiters.
   VCP is transport-agnostic, blockchain-agnostic, and model-agnostic.
   Any system implementing VCP is an Epistemic Execution System (EES).

   The key invariant: C(manipulation) > C(validation) for all Claim
   objects.

1. Introduction
   As artificial intelligence transitions to autonomous agents capable
   of independent actions, the lack of a standardized mechanism for
   verifiable trust and epistemic convergence has become a critical
   bottleneck. VCP addresses this by providing a minimal protocol for
   claims, actions, and verifiable traces.

   The key invariant is that the cost of manipulating a Claim's
   confidence exceeds the cost of validating it for all Claim objects:
   C(manipulation) > C(validation).

2. Conformance
   A system is VCP-Compatible iff:

   1. It implements the 4 Core Endpoints in Section 4.
   2. confidence for all Claims is computed via Multiplicative Weights
      Update + Gossip Consensus as defined in Section 5 (GQRVP).
   3. All Action objects reference Claim[] as basis. Actions without
      basis MUST be rejected at Gateway.
   4. Trace is immutable, content-addressed, and publicly auditable.

3. Core Types

3.1. Claim
   The atomic unit of verifiable information.

   {
     "claim_id": "sha256:...",
     "content": "string",
     "provenance": [{"type": "url|attestation", "src": "string",
                     "signed_by": "did:..."}],
     "method": "string",
     "confidence": "float 0.0-1.0",
     "created_at": "RFC3339 timestamp",
     "expires": "RFC3339 timestamp",
     "trace": ["claim_id"]
   }

3.2. Formal Completeness Boundary
   VCP operates at the boundary of formal provability. Formal-to-
   epistemic transition threshold: theta_critical = 0.731. When
   confidence >= threshold, system may treat result as formally
   sufficient. When below, SHOULD escalate to Continuum-style semantic
   verification.

   Empirical sybil-10x-g1.5 trace: 30 confidence scores extracted,
   theta_critical 0.731 (knee point), 0 below, 30 above. Consistent
   with Godel-grounded design.

3.3. Attestation
   Cryptographic signature over a Claim by a DID.

   {
     "attestation_id": "sha256:...",
     "claim_id": "sha256:...",
     "issuer": "did:...",
     "signature": "bytes"
   }

3.4. Action
   A proposed state change grounded in Claims.

   {
     "action_id": "sha256:...",
     "name": "string",
     "params": {},
     "basis": ["claim_id"],
     "stake": "uint",
     "proposed_by": "did:..."
   }

3.5. Trace
   Immutable record of verification.

   {
     "trace_id": "sha256:...",
     "action_id": "sha256:...",
     "result": "ACCEPTED|REJECTED",
     "reason": "string",
     "final_confidence": "float",
     "gqrvp_rounds": "uint",
     "timestamp": "RFC3339 timestamp",
     "merkle_root": "bytes"
   }

4. Core Endpoints
   All endpoints MUST be served over HTTPS. Request/response bodies are
   application/json.

4.1. POST /v1/claim
   Submit new Claim. Request: Claim without claim_id, confidence.
   Response 201: {claim_id}. 400: Invalid schema.

4.2. GET /v1/claim/{claim_id}
   Retrieve Claim and current confidence.
   200: {claim, confidence, attestations, merkle_proof}. 404: Not found.

4.3. POST /v1/action/verify
   Verify Action. Request: Action. Response 200:
   {status ACCEPTED|REJECTED, trace_id, reason}.
   Gateway MUST reject if basis empty or confidence < 0.731.

4.4. GET /v1/trace/{trace_id}
   Retrieve immutable history. 200: Trace. 404: Not found.

5. GQRVP: Security Parameters
   To prevent Sybil and amplification attacks, VCP-Compatible
   implementations MUST use:

   Parameter      Symbol   Value   Purpose
   -----------------------------------------------
   Learning Rate  eta      0.2     Controls weight update speed
   Amplification  gamma    1.5     Amplifies consensus
   Mixing         eps      0.1     Prevents Sybil isolation

   Formal proof of 67% Byzantine resilience under these parameters is
   given in /specs/RFC/GQRVP-security.md.

7. Security Considerations
   VCP does not resolve conflicts caused by asymmetry of goals. VCP
   guarantees that cost of manipulating a Claim's confidence exceeds
   cost of validating it, given honest supermajority of stake-weighted
   attention.

Appendix B. Godel and theta_critical
   Threshold theta_critical=0.731 is practical boundary between formal
   sufficiency and semantic escalation, grounded in Godel's Second
   Incompleteness Theorem. In sybil-10x-g1.5 trace: 30 scores, 0 below,
   30 above threshold. Demonstrates stable high-confidence convergence.
